1.1

Introduction to NumPy

🧠
When to use: Whenever you need to work with numbers in bulk — lists of values, tables of data, image pixels, math computations. It's far faster than plain Python lists.
NumPy is the backbone of almost all Python data work. Its core object — the ndarray — is a fast, multi-dimensional container for numbers. Unlike a plain Python list, every element shares the same type, which makes operations dramatically faster. Think of it as the spreadsheet grid that lives under pandas, scikit-learn, and TensorFlow — you'll be using it every time you touch a dataset.

NumPy's core object is the ndarray (n-dimensional array). Think of it as a supercharged list that can hold numbers in any shape — 1D (a row), 2D (a table), or 3D (a cube).

python
import numpy as np

# 1D array — a flat list of numbers
a = np.array([1, 2, 3, 4])

# 2D array — a matrix (rows × columns)
m = np.array([[1, 2], [3, 4]])

# Useful info about your array
a.shape       # → (4,)    how many in each dimension
m.shape       # → (2, 2)  rows, columns
a.dtype       # → int64   what type of numbers
a.ndim        # → 1       how many dimensions
💡 Memory trick: shape reads like coordinates — (rows, columns). A (3, 4) array has 3 rows and 4 columns.
Ready-made arrays
np.zeros((3,3))
np.ones((2,4))
np.eye(3)
np.full((2,2), 7)
Range arrays
np.arange(0, 10, 2)
→ [0, 2, 4, 6, 8]
np.linspace(0, 1, 5)
→ [0, .25, .5, .75, 1]
⚡
arange — like Python's range() but returns a NumPy array. Use when you need a range with a fixed step size.
⚡
linspace — give me exactly N evenly spaced points between A and B. Use when you care about the number of points, not the step size.

NumPy Functions

🧠
When to use: Any time you need to summarize, transform, or reshape your data — without writing loops.
These are the everyday workhorses of NumPy. Instead of looping through thousands of rows to calculate an average, you just call np.mean() and it's done in one line. Reshaping lets you reorganize data into the shape a model or function expects — a critical skill when feeding data into machine learning pipelines.

a = np.array([[1,2,3],[4,5,6]])  # shape (2,3)

np.sum(a)               # → 21  (all elements)
np.sum(a, axis=0)       # → [5,7,9]  column sums
np.sum(a, axis=1)       # → [6,15]   row sums

a.reshape(3, 2)         # 3 rows, 2 cols (same 6 elements)
a.T                     # shape becomes (3,2)
💡 axis=0 moves DOWN the rows (gives one result per column). axis=1 moves ACROSS the columns (gives one result per row).

Trigonometric Functions & Arithmetic Operations

🧠
When to use: Math on entire arrays at once — signals, angles, element-by-element math. NumPy does the loop for you.
This is where NumPy really shines as a math engine. Whether you're computing sine waves for signal processing, squaring values for loss functions, or rounding predictions to two decimals — all of it happens across the entire array in one call. No for-loops, no manual indexing, just clean vectorized math.
Trig
np.sin(a)
np.cos(a)
np.tan(a)
np.deg2rad(180)
np.rad2deg(np.pi)
Exponents & Logs
np.exp(a)
np.log(a)
np.log2(a)
np.log10(a)
np.sqrt(a)
Rounding
np.round(a, 2)
np.floor(a)
np.ceil(a)
np.abs(a)
python
a = np.array([0, 30, 60, 90])
angles = np.deg2rad(a)       # convert degrees → radians first!
np.sin(angles)              # → [0, 0.5, 0.866, 1.0]

# Arithmetic works element-by-element
x = np.array([1, 2, 3])
y = np.array([4, 5, 6])
x + y                       # → [5, 7, 9]
x * y                       # → [4, 10, 18]
x ** 2                      # → [1, 4, 9]
⚡
NumPy trig functions expect radians, not degrees. Always convert with np.deg2rad() first — forgetting this is one of the most common NumPy mistakes.
1.4

Operations on Matrices

🧠
When to use: Linear algebra, machine learning math, solving systems of equations, transformations.
Matrices are how data science thinks about data — rows are samples, columns are features. Once you get into ML, you'll see matrix operations everywhere: weights multiplied by inputs, covariance matrices, dimensionality reduction. Understanding A @ B vs A * B is one of the most important distinctions in all of NumPy.
python
A = np.array([[1,2],[3,4]])
B = np.array([[5,6],[7,8]])

# Element-wise vs matrix multiplication
A * B                       # element-wise: [[5,12],[21,32]]
A @ B                       # matrix multiply: [[19,22],[43,50]]
np.dot(A, B)               # same as A @ B

# Linear algebra operations
np.linalg.det(A)           # determinant → -2.0
np.linalg.inv(A)           # inverse matrix
np.linalg.eig(A)           # eigenvalues and eigenvectors
A.T                         # transpose
💡 Key distinction: A * B multiplies matching positions. A @ B does true matrix multiplication (rows × columns). In data science, you almost always want @.

1.5

Random Function in NumPy

🧠
When to use: Simulations, generating test data, initializing weights in ML models, shuffling datasets, sampling.
Randomness is everywhere in data science — from train/test splits to dropout in neural networks. NumPy's random module gives you controlled, reproducible randomness. The key word is controlled: by setting a seed you get the same "random" numbers every run, which means your experiments are repeatable and shareable.
python
rng = np.random.default_rng(42)   # seed=42 → reproducible results

# Generate random numbers
rng.random(5)               # 5 floats between 0.0 and 1.0
rng.integers(1, 10, 5)     # 5 random integers between 1–9
rng.normal(0, 1, (3,3))    # 3×3 from normal dist (mean=0, std=1)

# Useful utilities
a = np.array([1,2,3,4,5])
rng.shuffle(a)              # shuffle in-place
rng.choice(a, 3)            # pick 3 random items
Distributions
rng.random()
rng.normal(μ, σ)
rng.uniform(lo, hi)
rng.binomial(n, p)
Sampling
rng.integers(lo,hi,n)
rng.choice(array, n)
rng.shuffle(array)
rng.permutation(n)
⚡
Always set a seed (e.g., default_rng(42)) when you want the same random numbers every run. Critical for reproducible experiments — without it, results change every time.
1.6

Accessing NumPy Arrays

🧠
When to use: Extracting specific rows, columns, elements, or sub-sections of your data.
Indexing and slicing is how you talk to your data. Want the first 100 rows? Grab every other column? Filter out all values below zero? This is the tool. Mastering slicing — especially boolean indexing — is what separates someone who can read NumPy code from someone who can actually write it confidently.
python
a = np.array([[10,20,30],
               [40,50,60],
               [70,80,90]])

# Single element: [row, col]
a[0, 2]          # → 30  (row 0, col 2)
a[-1, -1]        # → 90  (last row, last col)

# Slicing: [start:stop:step]  (stop is NOT included)
a[0, :]           # → [10, 20, 30]  entire row 0
a[:, 1]           # → [20, 50, 80]  entire column 1
a[0:2, 1:3]       # → [[20,30],[50,60]]

# Boolean (conditional) indexing
a[a > 50]          # → [60, 70, 80, 90]
a[a % 20 == 0]    # → all multiples of 20

# Fancy indexing
a[[0, 2], :]       # → rows 0 and 2 only
💡 Slicing rule: [row_slice, col_slice]. The colon : means "all". Think of it like reading a table: row first, then column.
⚡
Slices return a view, not a copy — changing it changes the original array too. Use .copy() if you need an independent version: b = a[0:2].copy()
1.7

Modifying the Entries of a Matrix

🧠
When to use: Updating values in-place, replacing bad data, transforming specific cells, combining arrays.
Real-world data is messy — negative values where there shouldn't be any, missing entries, data from two sources that needs merging. This section covers how to clean and reshape data directly in NumPy. np.where() in particular is a must-know: it's a one-liner that replaces entire loops of if-else logic.
python
a = np.array([[1,2,3],[4,5,6]])

# Modify a single element
a[0, 1] = 99          # → [[1,99,3],[4,5,6]]

# Modify an entire row or column
a[1, :] = 0            # set all of row 1 to zero
a[:, 0] = [10, 20]    # set column 0

# Conditional modification
a[a < 0] = 0           # clip negatives to zero
np.where(a > 3, a, 0) # keep values >3, else replace with 0

# Adding / removing rows & columns
np.append(a, [[7,8,9]], axis=0)  # add a row
np.delete(a, 1, axis=0)          # delete row 1
np.concatenate([a, b], axis=0)   # stack vertically
np.concatenate([a, b], axis=1)   # stack horizontally
💡 np.where(condition, if_true, if_false) — a vectorized if-else that runs on the entire array at once.
1.8

Saving and Loading NumPy Arrays

🧠
When to use: Persisting processed data so you don't have to recompute it. Great for large datasets or trained model weights.
Processing a large dataset can take minutes or hours. Saving the result means you only do it once — next time you just load the file and pick up where you left off. This is a habit every data scientist develops early. The .npy format is NumPy's native binary format — it's fast, lossless, and preserves your array's exact shape and data type.
python
a = np.array([[1,2],[3,4]])

# .npy — single array, binary format (fast)
np.save('my_array.npy', a)
b = np.load('my_array.npy')

# .npz — MULTIPLE arrays, access by key
np.savez('data.npz', x=a, y=b)
loaded = np.load('data.npz')
loaded['x']                       # access by key

# .csv — human-readable, shareable
np.savetxt('data.csv', a, delimiter=',')
c = np.loadtxt('data.csv', delimiter=',')
